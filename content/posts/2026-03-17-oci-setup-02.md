현재 생성한 인스턴스는 Ubuntu 24.04 Minimal 버전으로, 기본 패키지와 도구가 생략되어 있어 초기 설정이 필요합니다.  
특히 SSH 포트 변경과 방화벽 설정에서 막히는 경우가 많기 때문에, 제가 최종적으로 적용한 과정을 정리했습니다.

## 1. 시스템 패키지 업데이트 및 자동 보안 업데이트

시스템을 최신 상태로 유지하고, 보안 업데이트를 자동화합니다.

```bash
sudo apt update && sudo apt upgrade -y

# 자동 보안 업데이트
sudo apt install unattended-upgrades -y
sudo dpkg-reconfigure unattended-upgrades
```

## 2. 시간 및 로케일 설정

서버 로그와 시스템 환경을 위해 타임존과 로케일을 설정합니다.

```bash
# 로케일 설정
sudo locale-gen en_US.UTF-8 ko_KR.UTF-8
sudo update-locale LANG=en_US.UTF-8 LC_MESSAGES=en_US.UTF-8

# 타임존을 한국 표준시(KST)로 변경
sudo timedatectl set-timezone Asia/Seoul

# 확인
timedatectl
```

## 3. Swap 메모리 설정

OCI Free Tier ARM 인스턴스는 기본적으로 메모리가 충분하지만, 예기치 않은 상황에서 **Out Of Memory(OOM)** 로 시스템이 멈추는 것을 방지하기 위해 소량의 Swap 공간을 추가했습니다.

```bash
# Swap 설정
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 재부팅 시 자동 적용
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# Swap 적용 확인
free -h
```

## 4. 방화벽 설정 (UFW)

Ubuntu 기본 방화벽(UFW)을 활성화하고 필수 포트를 허용합니다.

```bash
sudo apt install ufw -y

# 기본 정책 설정
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 필수 포트 허용
sudo ufw allow 20022/tcp   # SSH 커스텀 포트
sudo ufw allow 80/tcp      # HTTP
sudo ufw allow 443/tcp     # HTTPS

# 방화벽 활성화
sudo ufw enable
```

## 5. SSH 보안 설정

SSH 기본 보안을 강화합니다.

```bash
sudo nano /etc/ssh/sshd_config

# 아래 항목 수정
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes

# SSH 포트
Port 22
Port 20022

# 설정 적용
sudo systemctl restart ssh
```

> ⚠️ SSH 키 인증이 정상 작동하는지 확인한 후 진행해야 합니다.

## 6. Fail2ban 설치 (SSH 공격 방지)
인터넷에 노출된 서버는 **무차별 대입 공격(Brute Force)** 을 지속적으로 받습니다.
이를 방지하기 위해 `fail2ban`을 설치합니다.

```bash
sudo apt install fail2ban -y

# 설정 파일 설정
sudo nano /etc/fail2ban/jail.local

# 서비스 상태 확인
sudo systemctl status fail2ban
```

## 7. SSH 포트 변경 시 발생한 접속 문제
SSH 포트를 변경한 후 외부에서 접속이 되지 않는 문제가 발생했습니다.

원인을 찾기 위해 확인해보니 Ubuntu 24.04 Minimal 환경에서는 SSH가 **socket 기반 활성화(socket activation)** 방식으로 동작하고 있었습니다.

즉 `ssh.service`가 직접 포트를 여는 것이 아니라 `ssh.socket`이 포트를 열고 연결 요청이 들어오면 `sshd`를 실행합니다.

구조는 다음과 같습니다.

```
ssh.socket
↓  
ssh.service
↓  
sshd
```

따라서 SSH 설정을 변경한 후에는 `ssh.service`가 아니라 `ssh.socket`을 재시작해야 포트 변경이 적용됩니다.

```bash
sudo systemctl daemon-reload
sudo systemctl restart ssh.socket

# 포트 적용 확인
sudo ss -tlnp | grep ssh
``` 

하지만 이 설정 이후에도 SSH 접속이 되지 않았습니다.

OCI 환경에서는 네트워크가 다음과 같은 다중 방화벽 구조를 가지기 때문에, 여러 지점을 함께 확인해야 합니다.

```
Internet  
↓  
OCI Security List / NSG  
↓  
Ubuntu UFW  
↓  
iptables  
↓  
ssh.socket  
↓  
sshd
```

```bash
sudo iptables -I INPUT 1 -p tcp --dport 20022 -j ACCEPT
```

OCI 콘솔에서 **VCN → Security List / NSG → Ingress Rule** 에 `TCP 20022` 포트를 추가했지만 여전히 접속이 되지 않았습니다.

결과적으로 `iptables` 규칙을 직접 추가하자 정상적으로 접속이 가능했습니다.

```bash
sudo iptables -I INPUT 1 -p tcp --dport 20022 -j ACCEPT
```

이는 UFW 설정과 iptables 규칙이 완전히 동기화되지 않은 상태에서 발생한 문제로 보입니다.
