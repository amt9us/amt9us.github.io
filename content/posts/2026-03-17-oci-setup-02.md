현재 생성한 인스턴스는 Ubuntu 24.04 Minimal 버전으로, 기본 패키지와 도구가 생략되어 있어 초기 설정이 필요합니다.  
특히 SSH 포트 변경과 방화벽 설정에서 막히는 경우가 많기 때문에, 제가 최종적으로 적용한 과정을 정리했습니다.

## 1. 시스템 복구 (sudo unminimize)
Minimal 이미지에서 생략된 man 페이지, 로컬라이징 파일, 기본 도구들을 복구하여 일반적인 Ubuntu 서버 환경으로 복구합니다.

```bash
sudo unminimize
```

> **주의:** 작업량이 많아 약 10~15분 정도 소요될 수 있습니다.

## 2. 패키지 업데이트 및 자동 보안 설정

시스템을 최신 상태로 유지하고, 보안 업데이트를 자동화합니다.

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install unattended-upgrades -y
sudo dpkg-reconfigure unattended-upgrades
```

## 3. 시간 및 로케일 설정

서버 로그와 시스템 환경을 위해 타임존과 로케일을 설정합니다.

```bash
# 타임존을 한국 표준시(KST)로 설정
sudo timedatectl set-timezone Asia/Seoul

# 확인
timedatectl

# 로케일 설정 (한글 깨짐 방지)
sudo locale-gen en_US.UTF-8
sudo update-locale LANG=en_US.UTF-8
```

## 4. Swap 메모리 설정 및 보안 도구 설치

OCI ARM 인스턴스는 RAM이 넉넉하지만(24GB 기준), 만약의 상황에서 OOM(Out Of Memory)으로 시스템이 멈추는 것을 방지하기 위해 스왑 공간을 생성합니다.  
또한 무차별 대입 공격(Brute Force)을 막기 위해 `fail2ban`을 설치합니다.

```bash
# Swap 설정
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 재부팅 시 자동 적용
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# Swap 적용 확인
free -h

# Fail2ban 설치
sudo apt install fail2ban -y
```

## 5. SSH 보안 설정

루트 로그인 차단과 비밀번호 로그인 비활성화를 통해 보안을 강화합니다.
> **주의:** SSH 키를 먼저 등록한 후 진행해야 합니다.

```bash
sudo vim /etc/ssh/sshd_config

# 아래 항목 수정
PermitRootLogin no
PasswordAuthentication no

# 커스텀 포트 사용 예시
Port 22          # 기본 포트
Port 20022       # 추가할 포트

# 변경 적용
sudo systemctl restart ssh
```

## 6. 방화벽 설정 (UFW)

Ubuntu 기본 방화벽(UFW)을 활성화하고 필수 포트를 허용합니다.

```bash
sudo apt install ufw -y

# SSH, HTTP, HTTPS 허용
sudo ufw allow 22/tcp
sudo ufw allow 20022/tcp   # 커스텀 포트
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 방화벽 활성화
sudo ufw enable
```

### 포트 변경 적용 트러블슈팅
Ubuntu 24.04부터 SSH가 `ssh.socket` 방식으로 관리되어 커스텀 포트가 바로 적용되지 않을 수 있습니다.  
이 경우 전통적인 서비스 방식으로 전환해야 합니다. 

```bash
# 소켓 비활성화 및 SSH 서비스 재시작
sudo systemctl stop ssh.socket
sudo systemctl disable ssh.socket
sudo systemctl daemon-reload
sudo systemctl restart ssh

# 포트 리스닝 확인
sudo ss -tlnp | grep ssh
```

> **주의:** OCI 콘솔에서 **VCN → Security Lists / NSG → Ingress Rules**에 TCP 20022 포트를 반드시 허용해야 외부 접속이 가능합니다.
