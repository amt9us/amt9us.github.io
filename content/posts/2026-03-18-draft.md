오라클 클라우드(OCI)에 Ubuntu 24.04 서버를 구축한 후, 이제 본격적으로 **개발 환경을 세팅**하려 합니다.

보통 서버에서는 `apt`로 패키지를 설치하지만, 개발 도구까지 전부 시스템 패키지로 관리하면 서버마다 환경이 달라지고 의존성이 충돌하는 문제가 생깁니다.

이를 해결하기 위해 이번에는 Nix 기반 개발 환경을 구축하기로 했습니다.  
Nix를 사용하면 서버 환경을 **코드처럼 관리**할 수 있고, 나중에 맥북이나 다른 서버에서도 **동일한 환경을 쉽게 재현**할 수 있습니다.

## 1. Nix를 사용하게 된 이유

### 기존 방식의 한계

전통적인 `apt` 방식은 패키지를 설치할 때 시스템 라이브러리를 공유하기 때문에 **의존성 충돌**이 발생할 수 있습니다.  
또한, 서버를 새로 만들 때마다 CLI 도구, 쉘 설정, 개발 환경 구성을 **매번 반복**해야 하는 번거로움이 있습니다.


### Nix란?

Nix는 **함수형 패키지 관리자**로, 모든 패키지를 불변(Immutable) 방식으로 관리합니다.  
`/nix/store/<hash>-package-name` 경로에 독립적으로 설치되고, 다음과 같은 강력한 특징이 있습니다.

- **독립성**: 시스템 기본 라이브러리를 건드리지 않아 서버가 항상 깨끗합니다.
- **재현성**: 동일한 설정이면 맥북이나 다른 서버에서도 1초 만에 같은 환경을 만듭니다.
- **롤백**: 업데이트가 잘못되어도 즉시 이전 상태로 되돌릴 수 있습니다.

### APT와 Nix 역할 분리

| 구분 | 패키지 매니저 | 대상 패키지 |
|:---:|:---:|:---|
| **OS / 시스템** | **APT** | `openssh`, `nginx`, `docker`, 커널 관련 패키지 |
| **개발 도구** | **Nix** | `git`, `zsh`, `starship`, 최신 CLI 도구 및 런타임 |

## 2. Nix 설치 및 시행착오 해결

현대적이고 안정적인 [Determinate Systems](https://install.determinate.systems/nix)의 설치러를 사용합니다.

```bash
# 설치 스크립트 실행
curl --proto '=https' --tlsv1.2 -sSf -L https://install.determinate.systems/nix | sh -s -- install
```

### 💡 시행착오: snapd 충돌 해결

설치 중 다음과 같은 에러가 발생할 수 있습니다.

```bash
stderr: /usr/lib/tmpfiles.d/snapd.conf:1: Unknown command type '!'.
```

이는 Ubuntu의 snapd 설정과 Nix 설치 프로그램 간의 충돌입니다.    

다음과 같은 방법으로 해결했습니다.

```bash
# 1. 문제가 되는 설정 잠시 이동
sudo mv /usr/lib/tmpfiles.d/snapd.conf /tmp/

# 2. Nix 설치 재실행 후 완료되면 복구
sudo mv /tmp/snapd.conf /usr/lib/tmpfiles.d/

# 3. Nix 환경 활성화
. /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh

# 4. 버전 체크
nix --version
```

## 3. Zsh & UX 환경 구축

기본 Bash 대신 macOS 표준이자 강력한 플러그인을 지원하는 `Zsh`와 `Starship`을 설치했습니다.

### Zsh 및 필수 플러그인 설치

```bash
nix profile add \
  nixpkgs#zsh \
  nixpkgs#zsh-autosuggestions \
  nixpkgs#zsh-syntax-highlighting \
  nixpkgs#zsh-completions
```

| 도구                     | 설명                           | 비고                     |
|--------------------------|--------------------------------|--------------------------|
| zsh-autosuggestions      | 명령어 자동 제안 플러그인        | 이전 입력 기반 추천      |
| zsh-syntax-highlighting  | 명령어 문법 하이라이트          | 입력 시 색상으로 구분    |
| zsh-completions          | 추가 자동완성 기능 제공          | 다양한 CLI 완성 지원     |

### .zshrc 설정

```bash
# 1. Nix 경로 설정
. /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh

# 2. 플러그인 연결 (Nix 설치 경로)
source ~/.nix-profile/share/zsh-autosuggestions/zsh-autosuggestions.zsh
source ~/.nix-profile/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh

# 3. 자동 완성 설정
autoload -Uz compinit && compinit
```

### 기본 쉘 변경 및 실행

```bash
# 기본 쉘 변경 (비밀번호 입력 필요)
sudo chsh -s $(which zsh) $USER

# 지금 바로 Zsh 실행 (혹은 재접속)
exec zsh
```

## 4. 현대적 CLI 도구 셋업

기존 리눅스 명령어를 대체하는 더 빠르고 예쁜 차세대 도구들입니다.

### ① 시스템 탐색 및 관리
```bash
nix profile add \
  nixpkgs#starship \
  nixpkgs#eza \
  nixpkgs#bat \
  nixpkgs#fd \
  nixpkgs#ripgrep \
  nixpkgs#zoxide \
  nixpkgs#btop \
  nixpkgs#duf \
  nixpkgs#ncdu \
  nixpkgs#tldr \
```

| 도구      | 설명                         | 비고                                   |
|-----------|------------------------------|----------------------------------------|
| starship  | 빠르고 가벼운 크로스쉘 프롬프트 | `eval "$(starship init zsh)"` 설정 필요 |
| eza       | 아이콘을 지원하는 `ls` 대체 도구 | 디렉토리 정렬 및 Git 상태 표시 지원     |
| bat       | 구문 강조를 지원하는 `cat` 대체 | 페이징 및 Git 변경 표시 지원            |
| fd        | 빠르고 직관적인 `find` 대체   | 간단한 문법과 빠른 검색 속도            |
| ripgrep   | 매우 빠른 `grep` 대체 검색 도구 | 코드 검색에 최적화 (`rg` 명령어 사용)   |
| zoxide    | 스마트한 `cd` 대체 도구      | 방문 기록 기반으로 `z` 명령어 이동      |
| btop      | 터미널 기반 시스템 모니터     | CPU, 메모리, 프로세스 시각화            |
| duf       | 디스크 사용량 확인 도구      | `df`보다 보기 쉬운 출력 제공             |
| ncdu      | 디렉토리 용량 분석 도구      | 폴더별 디스크 사용량 탐색               |
| tldr      | `man` 페이지 요약 도구       | 자주 사용하는 명령어 예제 제공           |

### .zshrc 설정

```bash
eval "$(starship init zsh)"
eval "$(zoxide init zsh)"
```

## 5. Git & GitHub 설정

### 도구 설치

```bash
nix profile add \
  nixpkgs#gh \
  nixpkgs#lazygit \
  nixpkgs#delta \
  nixpkgs#direnv
```

| 도구     | 설명                              | 비고                                      |
|----------|-----------------------------------|-------------------------------------------|
| gh       | GitHub CLI 도구                    | 터미널에서 PR, 이슈, repo 관리 가능       |
| lazygit  | 터미널 기반 Git UI                 | 직관적인 인터페이스로 Git 작업 수행        |
| delta    | Git diff 뷰어                      | 구문 강조 및 side-by-side diff 지원       |
| direnv   | 디렉토리별 환경 변수 관리 도구     | `.envrc` 파일 기반 자동 환경 설정         |

### Git Config 최적화

```bash
eval "$(direnv hook zsh)"

# Git의 기본 출력기를 delta로 설정
git config --global core.pager "delta"

# 대화형 모드에서도 delta 사용
git config --global interactive.diffFilter "delta --color-only"

# delta 세부 옵션 (줄 번호 표시, 옆으로 나란히 보기 등)
git config --global delta.navigate true    # n/N 키로 변경사항 이동
git config --global delta.light false      # 어두운 테마 사용 시
git config --global delta.side-by-side true # 왼쪽/오른쪽 나란히 보기 (화면이 넓을 때 강추!)
git config --global delta.line-numbers true # 줄 번호 표시

# merge 할 때도 delta 사용
git config --global merge.conflictstyle "zdiff3"

# 기본 동작 최적화
git config --global init.defaultBranch main
git config --global pull.rebase true
git config --global push.autoSetupRemote true

# 화려한 로그 출력 설정 (git lg)
git config --global alias.lg "log --graph --pretty=format:'%C(auto)%h %C(blue)%an %Creset%s %C(dim)%cr' --all"


git config --global help.autocorrect 30
git config --global rebase.autoStash true

# 탭 간격이나 기본 뷰 설정
gh config set prompt enabled
```

### GitHub CLI (gh) 인증 시행착오

서버(SSH) 환경에서는 브라우저를 띄울 수 없어 `gh auth login` 시 에러가 납니다.

1. `gh auth login` 실행 후 `Login with a web browser` 선택.

2. 터미널에 뜬 **8자리 코드** 복사.

3. 내 PC 브라우저에서 `github.com/login/device` 접속 후 코드 입력.

4. 인증 후 `gh auth setup-git`으로 권한 연동.
