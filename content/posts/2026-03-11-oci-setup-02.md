## 1. 인스턴스 생성 핵심 스펙

가장 먼저 "상시 무료" 배지를 확인하며 아래 사양으로 인스턴스를 생성했습니다.
* 이미지: Ubuntu 24.04 Minimal (aarch64)
* 구성: VM.Standard.A1.Flex (4 OCPU, 24GB RAM)
* 스토리지: 부트 볼륨 200GB
* 네트워크: 공용 IPv4 주소 할당 체크

## 2. 초기 접속 및 용량 확인

맥북 터미널에서 다운로드한 `.key` 파일을 이용해 첫 접속에 성공했습니다. 200GB 볼륨이 정상적으로 잡혔는지 확인하는 것이 필수입니다.
```bash
# 키 권한 수정 및 접속
chmod 400 ssh-key-xxxx.key
ssh -i ssh-key-xxxx.key ubuntu@<내-서버-IP>

# 용량 확인
df -h
```

## 3. 나만의 계정(amt9us) 생성 및 권한 부여

기본 계정인 ubuntu 대신 개인 계정을 생성하고, 비밀번호 없이 관리자 권한(sudo)을 쓸 수 있게 세팅했습니다.
```bash
# 계정 생성 (비밀번호는 나중에 설정)
sudo adduser amt9us --gecos "" --disabled-password
sudo usermod -aG sudo amt9us

# 비밀번호 없이 sudo 사용 설정
echo "amt9us ALL=(ALL) NOPASSWD:ALL" | sudo tee /etc/sudoers.d/amt9us
sudo chmod 0440 /etc/sudoers.d/amt9us
```

## 4. 하이브리드 접속 환경 구축 (Key + Password)

### 4.1 SSH 키 상속

ubuntu 계정이 가진 인증 키를 amt9us 계정으로 복사합니다.
```bash
sudo mkdir -p /home/amt9us/.ssh
sudo cp /home/ubuntu/.ssh/authorized_keys /home/amt9us/.ssh/
sudo chown -R amt9us:amt9us /home/amt9us/.ssh
sudo chmod 700 /home/amt9us/.ssh
sudo chmod 600 /home/amt9us/.ssh/authorized_keys
```

### 4.2 비밀번호 인증 활성화

Minimal 이미지라 편집기가 없을 땐 sed 명령어로 SSH 설정을 직접 수정합니다.
```bash
# 비밀번호 및 상호작용 인증 활성화
sudo sed -i 's/^#\?PasswordAuthentication .*/PasswordAuthentication yes/' /etc/ssh/sshd_config
sudo sed -i 's/^#\?KbdInteractiveAuthentication .*/KbdInteractiveAuthentication yes/' /etc/ssh/sshd_config
sudo systemctl restart ssh

# 계정 비밀번호 설정
sudo passwd amt9us
```
