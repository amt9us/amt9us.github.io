도커 컨테이너와 Nginx 같은 인프라 기술을 직접 다뤄보며 프로젝트를 배포해 보고자 개인 서버 구축을 결심했습니다.  
특정 사양 내에서 **'평생 무료'** 를 보장하는 **오라클 클라우드(OCI)** 를 선택했지만, 인스턴스 생성부터 환경 구축까지의 과정은 생각보다 꼼꼼한 설정이 필요했습니다.

## 1. 서버 구성: 왜 ARM Ampere 인스턴스인가?

OCI에서 제공하는 무료 인스턴스는 크게 AMD와 ARM 기반으로 나뉩니다.  
그중 **ARM 기반의 Ampere 인스턴스**는 무료라고 믿기 힘들 정도의 압도적인 사양을 제공합니다.

| **구분** | **AMD (Standard.E2.1.Micro)** | **ARM Ampere (A1.Flex)** |
| :--- | :--- | :--- |
| **CPU** | 1 OCPU (공유형) | **4 OCPU (전용)** |
| **RAM** | 1 GB | **24 GB** |
| **스토리지** | 기본 50 GB | **최대 200 GB (무료 범위 내)** |

다만, ARM 인스턴스는 높은 인기 때문에 발생하는 **"Out of capacity"** 오류가 고질적인 문제입니다.
  
저는 자리가 나길 기다리는 대신 **유료 계정으로 전환**하는 방법을 선택했습니다.  
프리티어 범위 내에서만 자원을 사용하면 실제로 비용이 청구되지 않으며, 전환 직후 그토록 안 잡히던 ARM 인스턴스를 즉시 생성할 수 있었습니다.

### 최종 생성 인스턴스 스펙
* **이미지:** Ubuntu 24.04 Minimal (aarch64)
* **구성:** VM.Standard.A1.Flex (4 OCPU, 24GB RAM)
* **스토리지:** 부트 볼륨 200GB (무료 최대치 적용)

## 2. 계정 설정 및 하이브리드 접속 환경 구축

기본 제공되는 `ubuntu` 계정 대신 개인 전용 계정을 생성하고, **"개인 PC에서는 키 파일로, 외부 기기에서는 비밀번호로"** 접속할 수 있는 유연한 하이브리드 환경을 구성했습니다.

### 2.1 개인 계정(amt9us) 생성 및 권한 부여
보안과 편의를 위해 개인 계정을 생성하고, `sudo` 명령어 사용 시 비밀번호 입력을 생략하도록 설정하여 작업 효율을 높였습니다.

```bash
# 계정 생성 및 sudo 그룹 추가
sudo adduser amt9us --gecos "" --disabled-password
sudo usermod -aG sudo amt9us

# 비밀번호 없이 sudo 사용 설정
echo "amt9us ALL=(ALL) NOPASSWD:ALL" | sudo tee /etc/sudoers.d/amt9us
sudo chmod 0440 /etc/sudoers.d/amt9us
```

### 2.2 SSH 키 상속

기존 `ubuntu` 계정에 등록된 공개키를 새 계정으로 복사하여, 기존에 다운로드한 `.key` 파일로 즉시 접속이 가능하게 설정합니다.
```bash
sudo mkdir -p /home/amt9us/.ssh
sudo cp /home/ubuntu/.ssh/authorized_keys /home/amt9us/.ssh/
sudo chown -R amt9us:amt9us /home/amt9us/.ssh
sudo chmod 700 /home/amt9us/.ssh
sudo chmod 600 /home/amt9us/.ssh/authorized_keys
```

### 2.3 비밀번호 인증 활성화

키 파일이 없는 외부 기기에서도 접속할 수 있도록 SSH 설정을 수정합니다.

현재 사용 중인 Minimal 버전 이미지에는 `vi`나 `nano` 같은 에디터가 기본 포함되어 있지 않거나 사용이 번거로울 수 있어, `sed` 명령어를 활용해 직접 설정을 치환했습니다.
```bash
# SSH 설정 수정: 비밀번호 및 상호작용 인증 활성화
sudo sed -i 's/^#\?PasswordAuthentication .*/PasswordAuthentication yes/' /etc/ssh/sshd_config
sudo sed -i 's/^#\?KbdInteractiveAuthentication .*/KbdInteractiveAuthentication yes/' /etc/ssh/sshd_config

# SSH 서비스 재시작하여 변경사항 적용
sudo systemctl restart ssh

# 외부 접속 시 사용할 계정 비밀번호 설정
sudo passwd amt9us
```
