최근 레거시 시스템 구조를 살펴보던 중 흥미로운 점을 발견했습니다.

여러 서비스가 각각 다른 데이터베이스를 사용하는 줄 알았는데, 실제로 확인해 보니 하나의 **Oracle Database** 인스턴스 안에서 스키마만 분리된 구조로 운영되고 있었습니다.

즉, 서비스마다 사용하는 스키마는 서로 다르지만 일부 테이블은 **동일한 데이터를 공유**하는 형태였습니다.

이 구조를 확인하면서 다음과 같은 궁금증이 생겼습니다.

- 현재처럼 스키마를 분리하는 구조의 장점은 무엇인지?
- 서비스별로 완전히 분리된 데이터베이스를 사용하는 것이 더 좋은 구조인지?
- 이러한 구조는 어떤 이유로 사용되는 것인지?

이번 글에서는 이러한 궁금증을 바탕으로 스키마 기반 구조의 특징과 활용 방식을 정리해보았습니다.

## Oracle에서 Schema란 무엇인가

Oracle에서 **Schema**는 데이터베이스 객체들을 논리적으로 구분하기 위한 단위입니다.    

Oracle의 특징 중 하나는 사용자(User)와 스키마가 동일한 개념이라는 점입니다.  
하나의 사용자 계정을 생성하면 해당 계정 이름으로 스키마가 생성됩니다.

예를 들어 다음과 같은 계정이 있다면

```
service_a
service_b
```

각각이 하나의 스키마가 되고, 테이블은 다음과 같이 스키마와 함께 표현됩니다.

```
service_a.user_table
service_b.user_table
```

이처럼 스키마를 사용하면 같은 데이터베이스 안에서도 객체들을 논리적으로 분리하여 관리할 수 있습니다.

## 왜 이런 구조를 사용할까?

### 1. 공통 데이터 공유

여러 서비스에서 공통으로 사용하는 데이터가 존재할 수 있습니다.

예를 들어

- 사용자 정보
- 설정 정보

같은 데이터를 서비스마다 따로 저장하면 데이터 중복과 불일치 문제가 발생할 수 있습니다.

그래서 하나의 스키마에 실제 테이블을 두고, 다른 스키마에서는 `synonym`을 통해 접근하도록 구성합니다.

### 2. 서비스 단위 구조 분리

스키마를 분리하면 서비스별 데이터 구조를 **독립적으로 관리**할 수 있습니다.

예를 들어, 구조를 그림으로 표현하면 다음과 같습니다.

```
common
 └ 공통 테이블

service_a
 ├ 서비스 A 전용 테이블
 └ 공통 테이블 synonym

service_b
 ├ 서비스 B 전용 테이블
 └ 공통 테이블 synonym
```

이 구조는 다음과 같은 장점이 있습니다.

1. 서비스별 테이블 관리 가능
2. 공통 데이터는 하나만 유지

### 3. 운영 및 권한 관리

스키마 단위로 권한을 관리할 수 있기 때문에, 서비스별 접근 권한을 분리할 수도 있습니다.

예를 들어 특정 스키마에는

- 조회 권한만 허용
- 일부 테이블 접근 제한

같은 방식으로 운영할 수 있습니다.

## 서비스별 데이터베이스를 분리하는 것이 더 좋을까?

처음 구조를 확인했을 때는 서비스마다 완전히 독립된 데이터베이스를 사용하는 것이 더 좋은 구조가 아닐까라는 생각이 들었습니다.

실제로 많은 시스템에서는 다음과 같은 구조를 사용하기도 합니다.

```
Service A → DB A
Service B → DB B
Service C → DB C
```

이 방식의 장점은 다음과 같습니다.

- 서비스 간 결합도 감소
- 장애 영향 범위 축소
- 확장성 향상

하지만 서비스 간 공통 데이터가 많거나, 운영 복잡도를 줄여야 하는 환경에서는 지금 구조인 **하나의 DB + 스키마 분리** 구조가 더 현실적인 선택이 될 수도 있습니다.

---

이번 글을 통해, 독립된 서비스라 하더라도 유저 정보나 공통 데이터는 동일 테이블을 사용하는 것이 효율적이라는 점을 확인할 수 있었습니다.
중복 테이블 없이 데이터 일관성을 유지할 수 있어 운영과 유지보수가 쉬워집니다.

다만 현재 구조에도 개선 여지가 있습니다.

1. 테이블 접근 명확화
2. 장애 영향 최소화
3. 다른 DB 환경 대응

결론적으로, **Oracle**에서는 **스키마 분리 + 공통 테이블 Synonym 활용**이 서비스 독립성과 데이터 공유를 동시에 관리하는 현실적인 방법이라는 것을 알게되었습니다.
