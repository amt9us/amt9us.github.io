오라클 클라우드(OCI)에 Ubuntu 24.04 서버를 구축한 후, 이제 본격적으로 **개발 환경을 세팅**하려 합니다.

이번에는 Nix와 Home Manager를 활용해 **CLI 중심 개발 환경**을 선언적 방식으로 관리하며, 재현 가능하고 깨끗한 환경을 만드는 것을 목표로 했습니다.

## 1. Nix를 선택한 이유

기존 `apt` 기반 설치는 편리하지만, 서버마다 패키지 버전이 달라 의존성 충돌이 발생할 수 있습니다.  
Nix를 사용하면 다음과 같은 장점이 있습니다.

- **독립성**: `/nix/store` 아래 패키지가 설치되어 시스템 라이브러리와 격리됩니다
- **재현성**: 로컬이나 다른 서버에서도 동일한 환경을 쉽게 구성할 수 있습니다.
- **롤백**: 문제가 생기면 즉시 이전 상태로 되돌릴 수 있습니다.

### APT와 Nix 역할 구분

| 구분 | 패키지 매니저 | 대상 패키지 |
|:---:|:---:|:---|
| **OS / 시스템** | **APT** | `openssh`, `nginx`, `docker`, 커널 관련 패키지 |
| **개발 도구** | **Nix** | `git`, `zsh`, `starship`, CLI 도구 및 런타임 |

## 2. Nix 설치 및 초기 설정

```bash
# Nix 설치
curl --proto '=https' --tlsv1.2 -sSf -L https://install.determinate.systems/nix | sh -s -- install

# 설치 후 환경 활성화
. /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh

# 버전 확인
nix --version
```

> 💡 snapd 충돌 문제  
> Ubuntu 24.04 최소 설치 환경에서는 Snap 설정(`snapd.conf`)과 Nix 설치 스크립트가 충돌할 수 있습니다.
> ```bash
> sudo mv /usr/lib/tmpfiles.d/snapd.conf /tmp/
> # Nix 설치 재실행 후 복구
> sudo mv /tmp/snapd.conf /usr/lib/tmpfiles.d/
> ```

## 3. Home Manager + Zsh 환경 구축

Home Manager를 사용하면 패키지, 쉘 설정, alias, 플러그인까지 한 곳에서 선언적으로 관리할 수 있습니다.

최근에는 Home Manager + Flakes 조합이 권장됩니다.

### Flake 설정 활성화

```bash
mkdir -p ~/.config/nix
echo "experimental-features = nix-command flakes" >> ~/.config/nix/nix.conf
```

### Home-manager Flake 초기화

```bash
cd ~/.config/home-manager
nix flake init -t github:nix-community/home-manager
```

이후 `flake.nix`와 `home.nix`가 생성됩니다.

### Zsh 설치 및 기본 쉘 변경

```bash
sudo apt install zsh

# 로그인 쉘 변경
chsh -s /bin/zsh
```

## 4. Home Manager를 이용한 패키지 및 환경 설정

`home.nix` 구조

```nix
{ config, pkgs, ... }:

{
  home.username = "amt9us";
  home.homeDirectory = "/home/amt9us";
  home.stateVersion = "25.11";

  programs.zsh = {
    enable = true;
    enableCompletion = true;
    autosuggestion.enable = true;
    syntaxHighlighting.enable = true;

    envExtra = ''
      if [ -e /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh ]; then
        . /nix/var/nix/profiles/default/etc/profile.d/nix-daemon.sh
      fi
    '';
  };
}
```

설정 후에는 아래 명령으로 적용합니다.

```bash
home-manager switch --flake .
```

> Home Manager를 통해 Zsh 플러그인, 환경 변수, 패키지 설치까지 한 번에 관리할 수 있습니다.

## 5. CLI 생산성 도구 설치

### ① 시스템 탐색 및 관리

```nix
programs.starship.enable = true;
programs.zoxide.enable = true;

xdg.configFile."starship.toml".source = ./starship.toml;

home.packages = with pkgs; [
  eza
  bat
  fd
  ripgrep
  zoxide
  btop
  duf
  ncdu
  tldr
];
```

| 도구       | 설명                  |
| -------- | ------------------- |
| starship | 빠르고 가벼운 쉘 프롬프트      |
| zoxide   | 방문 기록 기반 빠른 디렉토리 이동 |
| eza      | 아이콘 지원 `ls` 대체 도구   |
| bat      | 구문 강조 `cat` 대체      |
| fd       | 빠른 `find` 대체        |
| ripgrep  | 고속 코드 검색 (`rg`)     |
| btop     | 터미널 기반 시스템 모니터      |
| duf      | 디스크 사용량 확인          |
| ncdu     | 디렉토리 용량 분석          |
| tldr     | 명령어 요약 & 예제 제공      |


### ② 네트워크 및 서버 디버깅

## 6. Git & GitHub 설정

### Home Manager 패키지

```nix
home.packages = with pkgs; [
  gh
  lazygit
  delta
];
```

| 도구      | 설명          |
| ------- | ----------- |
| gh      | GitHub CLI  |
| lazygit | 터미널 Git UI  |
| delta   | Git diff 뷰어 |

### Git / delta 설정

```nix
  programs.delta = {
    enable = true;
    enableGitIntegration = true;

    options = {
      navigate = true;
      side-by-side = true;
      line-numbers = true;
    };
  };

  programs.git = {
    enable = true;
    settings = {
      user = {
        name = "username";
        email = "l.com";
      };
      merge.conflictstyle = "zdiff3";
      init.defaultBranch = "main";
      pull.rebase = true;
      push.autoSetupRemote = true;
      help.autocorrect = 30;
      rebase.autoStash = true;
      alias = {
        lg = "log --graph --pretty=format:'%C(auto)%h %C(blue)%an %Creset%s %C(dim)%cr' --all";
      };
    };
  };
```

### GitHub CLI 인증 (SSH 서버 환경)

1. `gh auth login` → `Login with a web browser` 선택.

2. 터미널에 제공되는 **8자리 코드** 복사.

3. PC 브라우저에서 `github.com/login/device` → 코드 입력.

4. 인증 후 `gh auth setup-git`으로 실행.
